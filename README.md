# ScreenBridge

A free, no-friction, peer-to-peer screen-mirroring app for **macOS ⇄ Windows on
the same LAN**. Either device can act as **host** (shares its screen + system
audio) or **viewer** (shows the host's screen in a normal desktop window that is
itself capturable by Discord/OBS, so you can re-share it into a call).

> **Status: Phase 0 (foundations & scaffolding) complete.** The build skeleton,
> shared Rust core, FFI boundary, and both native app shells are in place and
> wired to CI. There is no capture, encode, transport, or pairing yet — those
> arrive in later phases. See [`plans/PLAN.md`](plans/PLAN.md) for the full
> specification and phase plan.

## Goals (v1)

- **Simple** — install → live stream in under 60 s; mDNS auto-discovery, 6-digit
  PIN pairing, zero config files.
- **Performant** — 1080p60 at ≤100 ms median glass-to-glass latency, fully
  hardware-accelerated, adaptive quality.
- **Secure** — TLS 1.3 (via QUIC) always on, SPAKE2 PIN pairing (MITM-resistant),
  view-only, no telemetry, no admin/root.
- **Discord-capturable** — the viewer window is enumerable and shows live content
  in Discord and OBS window-capture on both OSes.

## Architecture (at a glance)

A **shared Rust core** owns the wire (QUIC transport, framing, mDNS discovery,
SPAKE2 pairing) and exposes a tiny **C ABI**. **Native apps** own the hardware
path (capture, encode/decode, render) and the UI. The **encoded bitstream** is
the only thing that crosses the FFI boundary — Rust never touches raw pixels or
PCM, native never touches the socket. See [`docs/architecture.md`](docs/architecture.md).

```
core/        Rust workspace (transport, session, security, discovery, protocol, ffi)
include/     screenbridge.h — C ABI header, generated by cbindgen
apps/macos/  SwiftUI app shell (SPM) + ScreenCaptureKit/VideoToolbox path (later)
apps/windows/ WinUI 3 app shell + cross-platform P/Invoke interop
docs/        architecture, protocol, security model, build instructions
```

## Building

See [`docs/build.md`](docs/build.md) for prerequisites and full instructions. In
short:

```sh
# Rust core + generated C ABI header (include/screenbridge.h)
cargo build --manifest-path core/Cargo.toml

# macOS app shell + Swift FFI ping test
swift build --package-path apps/macos
swift test  --package-path apps/macos

# C# FFI ping test (runs against the native lib on any OS)
dotnet test apps/windows/ScreenBridge.Interop.Tests/ScreenBridge.Interop.Tests.csproj
```

The Windows WinUI 3 app shell builds on Windows (or CI's `windows-latest`); see
`docs/build.md`.
