# ScreenBridge — Wire Protocol

> **This document is the authoritative wire specification.** It is intentionally
> a skeleton at Phase 0 and is filled in alongside the transport (Phase 2). The
> Rust source of truth for the constants below is the `screenbridge-protocol`
> crate. Product context: [`../plans/PLAN.md`](../plans/PLAN.md) §6.

## Versioning

- Wire-protocol major version: **1** (`screenbridge_protocol::PROTOCOL_VERSION`).
- Capabilities are negotiated at session start via `Hello`. **Unknown major
  versions are rejected.**

## QUIC setup

- **ALPN:** `screenbridge/1` (`screenbridge_protocol::ALPN`). One QUIC connection
  per session.
- **Encryption:** TLS 1.3 via QUIC, always on. There is no plaintext mode.
- **Control:** one reliable bidirectional stream (length-prefixed messages).
- **Media:** unreliable **datagrams**, each ≤ path-MTU, carrying one fragment of
  an encoded frame. Delta video frames are never sent over a reliable stream
  (PLAN.md §10 R8).

## Media frame header (per datagram) — *Phase 2, illustrative*

```
track:      u8      // 0=video, 1=audio
codec:      u8      // 0=h264, 1=opus
frame_id:   u32     // monotonic per track
timestamp:  u64     // microseconds, shared clock for A/V sync
flags:      u8      // bit0=keyframe, bit1=frame_end
frag_index: u16
frag_count: u16
payload:    bytes
```

The receiver reassembles by `frame_id`, drops a frame if a fragment is missing
past a deadline (and requests a keyframe), and never blocks newer frames on
older ones. Reassembly buffers are bounded (PLAN.md §7).

## Control messages — *Phase 2/3, illustrative*

`Hello{version,role,capabilities}` · `PairBegin/PairMsg{spake2}` · `PairResult` ·
`StreamStart{codec,resolution,fps,audio}` · `KeyframeRequest` ·
`BitrateHint{target_kbps}` · `Pause/Resume` · `Stats{rtt,loss,fps,bitrate}` ·
`Bye{reason}`.

## C ABI surface

The native↔core boundary is documented in `include/screenbridge.h` (generated by
cbindgen from `core/crates/ffi`). Phase 0 exposes only the trivial liveness
surface: `sb_ping`, `sb_protocol_version`, `sb_version`. The media/session ABI
(`sb_init`, `sb_start_host`, `sb_send_encoded`, callbacks, …) is added per
PLAN.md §6.4 in later phases. All entry points are panic-safe (`catch_unwind`)
and document buffer ownership.
